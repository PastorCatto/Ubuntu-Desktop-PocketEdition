# Mobuntu
**Document Version 5.0 ‚Äî RC15 "The Debos Update"**

> Co-developed with Claude (Anthropic) and arkadin91 (reference image, firmware discovery, OTA script logic, cmdline tuning). Build scripts are intentionally readable and modular for long-term maintainability.

---

Mobuntu is a toolkit that builds a full Ubuntu ARM64 image for SDM845-based phones and Nintendo Switch devices. The end goal is to support the full range of devices Mobian targets, running a real Ubuntu release with a touch-first mobile interface.

No postmarketOS. No pmbootstrap. Everything is built via a debos recipe pipeline inside a debootstrap rootfs, then sealed and flashed via fastboot or Hekate.

## Architecture

### The Build Pipeline

```
1_preflight.sh       ‚Äî host deps, device/UI/release picker, build.env, generates run_build.sh
run_build.sh         ‚Äî generated ‚Äî calls debos for base tarball (cached) then device tarball
5_seal_rootfs.sh     ‚Äî unpacks device tarball, UUID, cmdline, fstab, boot.img or L4T files
verify_build.sh      ‚Äî unpacks and inspects device tarball before sealing
4_enter_chroot.sh    ‚Äî debug utility, drops you into the rootfs interactively
watchdog.sh          ‚Äî unattended runner, wraps run_build.sh
```

### Device Config System

Every device-specific setting lives in `devices/<brand>-<codename>.conf`. Script 1 presents a menu of available configs. Everything flows into `build.env` and `run_build.sh`. Key fields:

- `DEVICE_QUIRKS` ‚Äî space-separated flags controlling build behaviour:
  - `dtb_append` ‚Äî cat kernel + DTB before mkbootimg (required for SDM845)
  - `qcom_services` ‚Äî install and enable Qualcomm userspace daemons
  - `firmware_source_local` ‚Äî prompt for local firmware bundle
  - `firmware_source_online` ‚Äî pull firmware from apt
  - `l4t_bootfiles` ‚Äî output kernel.lz4 + initrd.lz4 + DTB (Switch)
- Kernel method and repo
- Boot method (`mkbootimg` / `l4t` / `uboot` / `uefi`) and all parameters
- Firmware method (`git` / `apt`) and repo URL
- DTB filename, mkbootimg offsets
- Device-specific apt packages and systemd services
- Hostname and image label

### debos Recipe Pipeline

`1_preflight.sh` generates `run_build.sh` which calls debos twice:

1. **Base build** (runs once, result cached as `base-{release}.tar.gz`):
   `base.yaml` ‚Üí debootstrap ‚Üí apt (base packages) ‚Üí UI ‚Üí user ‚Üí pack

2. **Device build** (per device, starts from unpacked base):
   `devices/{codename}.yaml` ‚Üí unpack base ‚Üí qcom.yaml or l4t.yaml ‚Üí kernel ‚Üí hostname ‚Üí pack ‚Üí `{label}-{release}.tar.gz`

`5_seal_rootfs.sh` unpacks the device tarball, writes UUID/cmdline/fstab, builds boot.img or L4T files, and creates the final rootfs image.

The base tarball is only rebuilt if `base.yaml` is newer than the existing tarball. Multi-device builds reuse the base ‚Äî 40-60% faster than the RC14 bash pipeline.

### fakemachine Backend

debos runs inside a VM for reproducibility. Backend is auto-detected:

| Backend | Speed | Requirement |
| :--- | :--- | :--- |
| kvm | ~9 min | `/dev/kvm` accessible |
| uml | ~18 min | `user-mode-linux` package |
| qemu | ~2.5 hr | `qemu-system-x86` (always available) |
| none | fastest | Root required, WSL2-safe |

WSL2 users: use `--disable-fakemachine` (override to `none` when preflight asks).

### Firmware Strategy

**SDM845 devices:**

1. **Local archive** ‚Äî `firmware/<brand>-<codename>/firmware.tar.gz` ‚Äî prompted at build time, applied as base layer
2. **Git clone** ‚Äî overlaid on top of local bundle
3. **Git-fail fallback** ‚Äî local bundle applied if clone fails
4. **OnePlus 6 apt fallback** ‚Äî last resort, with warning. GPU/WiFi/BT work; modem not guaranteed.

**OnePlus 6 / 6T:** All blobs ship in `linux-firmware` apt package ‚Äî no local bundle needed.

**Switch:** Firmware from apt / switchroot repos ‚Äî no local bundle.

**Adreno 630 GPU firmware** (`a630_sqe.fw`, `a630_gmu.bin`) always fetched from kernel.org.

### Audio

Audio works via PipeWire + ALSA UCM2 maps. `alsa-ucm-conf` is installed pinned to the Ubuntu release to prevent the Mobian repo from pulling an incompatible version. The firmware archive re-application after apt ensures UCM maps always win. `alsa-state` and `alsa-restore` are masked ‚Äî they conflict with the SDM845 audio subsystem.

**OnePlus 6/6T note:** `api.alsa.period-*` values are omitted from their WirePlumber config ‚Äî wireplumber auto-selection works better on enchilada/fajita (per upstream qcom-phone-utils 0.4.1 fix).

### Boot Cmdline (RC15, arkadin91)

```
root=UUID=${BUILD_UUID} earlycon console=tty0 console=ttyMSM0,115200 init=/sbin/init ro loglevel=7
```

Written to `/etc/kernel/cmdline` by `5_seal_rootfs.sh`. The active DTB is stored in `/etc/kernel/boot_dtb`.

### OTA Boot Updates

`apt upgrade` automatically rebuilds `boot.img` after every kernel update via:

- `/etc/kernel/postinst.d/zz-qcom-bootimg` ‚Äî fires after `apt install linux-image-*`
- `/etc/initramfs/post-update.d/bootimg` ‚Äî fires after `update-initramfs`

`qbootctl` is installed for slot updates without a PC.

## Build Workflow

### Prerequisites

- Ubuntu/Debian x86-64 or ARM64 host (native or WSL2)
- ~20GB free disk space
- Internet access
- WSL2 users: debos runs with `--disable-fakemachine` (select `none` for backend)

### Quick Start

```bash
# Make scripts executable
chmod +x *.sh *.py recipes/scripts/*.sh

# Run preflight ‚Äî installs host deps, picks device/UI/release, generates run_build.sh
sudo ./1_preflight.sh

# Build (generated by preflight)
sudo bash run_build.sh

# Verify before sealing (recommended)
bash verify_build.sh

# Seal and generate flash images
sudo bash 5_seal_rootfs.sh

# Optional: drop into the rootfs for debugging
sudo ./4_enter_chroot.sh
```

### Flashing (SDM845)

Boot into fastboot mode (Power + Volume Down), then:

```bash
fastboot flash boot   mobuntu_beryllium_<release>_boot.img
fastboot flash system mobuntu_beryllium_<release>_root_sparse.img
fastboot reboot
```

### Flashing (Switch)

Copy boot files to SD card, then flash rootfs via Hekate UMS:

```bash
# On SD card:
switchroot/ubuntu/kernel.lz4
switchroot/ubuntu/initrd.lz4
switchroot/ubuntu/<dtb>.dtb

# Rootfs via Hekate UMS or Flash Linux tool:
dd if=mobuntu_<codename>_<release>_root.img of=/dev/<linux-partition> bs=4M status=progress
```

## Adding a New Device

1. Create `devices/<brand>-<codename>.conf` using an existing config as a template
2. Create `recipes/devices/<codename>.yaml` using an existing device recipe as a template
3. Set `DEVICE_QUIRKS`, `FIRMWARE_METHOD`, `FIRMWARE_REPO`, `BOOT_DTB`, mkbootimg params, etc.
4. Optionally place `firmware.tar.gz` in `firmware/<brand>-<codename>/`
5. Run `1_preflight.sh` ‚Äî your device appears in the menu automatically

## Troubleshooting

**WSL2 / debos fakemachine errors:**
Select `none` when preflight asks for backend override. This runs `--disable-fakemachine` which requires sudo but works correctly in WSL2. Install `qemu-user-static` and run `sudo apt install --reinstall qemu-user-static` to register binfmt handlers for the ARM64 chroot stage.

**Emergency mode on boot:**
Run `systemctl --failed` and `cat /etc/fstab`. If `/boot/efi` appears, remove it: `sudo sed -i '/boot\/efi/d' /etc/fstab`.

**qcom_scm -22 error in dmesg:**
Cosmetic ‚Äî SLPI sensor hub crash loop. Does not affect display, audio, WiFi, modem or camera. See Known Issues. Do not install `hexagonrpcd` or `qcom-support-common`.

**No WiFi interface:**
Check `systemctl status tqftpserv` and `systemctl status rmtfs`. Confirm blobs at `/lib/firmware/qcom/sdm845/beryllium/`.

**No audio / dummy output:**
Confirm UCM2 maps at `/usr/share/alsa/ucm2/Xiaomi/beryllium/`. If missing, rebuild with a firmware archive that includes UCM maps.

**Modem crashes WiFi and BT:**
Known issue. Do not start ofono or ModemManager on beryllium until resolved.

**Random MAC address:**
Expected ‚Äî driver cannot read the encrypted `persist` partition, assigns random MAC for privacy.

**Black screen on boot:**
DTB mismatch. Confirm panel variant (Tianma vs EBBG) and use the correct device config. Check from TWRP: `cat /sys/class/graphics/fb0/modes`.

**Kernel hook not firing after apt upgrade:**
Check `/etc/kernel/postinst.d/zz-qcom-bootimg` is executable and `/etc/kernel/cmdline` exists with a valid UUID.

**Base tarball rebuilding every run:**
`base.yaml` is newer than `base-{release}.tar.gz`. Touch the tarball or delete and rebuild: the stale check is mtime-based.

## Known Issues

### SLPI crash loop (cosmetic)
- **Symptom:** `remoteproc2` crashes every 40s, `-22` SCM error in dmesg
- **Cause:** `sensor_process` user PD stalls without a FastRPC host daemon. `hexagonrpcd` requires droid-juicer vendor extraction not available in a clean Ubuntu build.
- **Impact:** Sensors non-functional. Everything else unaffected.
- **Fix:** None needed. Do not install `hexagonrpcd`.

### qcom-support-common -22 regression (Mobian upstream only)
- **Affects:** Mobian weekly builds post-March 2026. Mobuntu is not affected.
- **Cause:** `qcom-phone-utils 0.4.3` `qcom-firmware` hook rework introduced a beryllium path mismatch.
- **Fix:** Do not install `qcom-support-common` or `sdm845-support`.

### Switch KERNEL_REPO placeholder
- **Affects:** All four Switch device configs.
- **Fix:** Fill in `KERNEL_REPO` with the switchroot L4T kernel `.deb` URL before building. Verify DTB filenames against a working switchroot install.

### Modem crashes WiFi and BT
- Under investigation. Do not enable ModemManager or ofono on beryllium.
